## Native OAuth flow

### Login launch

`authGoogle()` now has two isolated branches:

- Native Capacitor:
  - `redirectTo:'ringgitme://auth/callback'`
  - `skipBrowserRedirect:true`
  - `prompt:'select_account'`
  - validates `data.url`
  - opens it with Capacitor Browser
- Non-native web/PWA:
  - retains `redirectTo:location.href.split('#')[0]`
  - retains the normal full-page Supabase redirect
  - does not access native plugins

### Callback and session restore

The native initializer runs once and supports:

- Warm return: `App.addListener('appUrlOpen', ...)`
- Cold start: `App.getLaunchUrl()`

The shared callback handler:

1. Accepts only protocol `ringgitme:`, host `auth`, and path `/callback`.
2. Reads `access_token` and `refresh_token` only from the URL hash fragment.
3. Queues an accepted URL only in private in-memory state if `supa` is not ready.
4. Calls `supa.auth.setSession({ access_token, refresh_token })` exactly once per accepted attempt.
5. Lets the existing `onAuthStateChange` handler perform `rememberAccount` and `adoptCloudOrPush`; it never calls cloud adoption directly.
6. Closes Capacitor Browser only after successful session restoration.
7. Shows generic login errors without exposing token details.

The implementation does not use or invoke `exchangeCodeForSession`; the project remains on the confirmed implicit OAuth flow.

## Duplicate and cancellation safety

- A single memoized initializer promise prevents repeated listener registration.
- In-memory `processing` and `handled` booleans suppress duplicate delivery from `getLaunchUrl`, `appUrlOpen`, or repeated iOS callbacks.
- No raw token or callback fragment is used as a persisted duplicate key.
- Starting a new native Google login resets only the temporary attempt state.
- Failure clears processing/pending state and the next explicit login attempt resets the handled guard.
- Canceling the Browser produces no callback, does not call `setSession`, does not alter the existing local session/data, and allows another login attempt.
- Browser closure is scoped to a successfully processed RinggitMe OAuth callback; unrelated Browser sessions are not force-closed.
